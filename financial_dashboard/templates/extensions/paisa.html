Automation & Caching

When auto sync is on, a coordinator coalesces dirtying core changes (new/edited transactions, accounts, cards, snapshots, CAS lots, or paisa.* settings) into one full-journal Paisa reload — there is no per-transaction/partial sync API, so a bulk statement import (one outer commit) is one bump and one coalesced reload. Fixed, non-tunable timings: 5s quiet debounce, 30s max dirty latency, 2s state polling, lease/single-flight, 1/2/5/10/15-min retry backoff, and a six-hour force reload. disabled/connect accumulate dirty state with no I/O; the min-reload interval is a hard floor between remote reloads and retries only (not the debounce).
